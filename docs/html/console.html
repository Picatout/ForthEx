Description

Fonctions associées à la console.
Système de console basé sur des tables de vecteurs pour chaque
type de terminal. les tables contiennent le CFA de chaque fonction.
Il y a 2 terminaux, la console LOCAL constituée du clavier et du moniteur
branché à l'ordinateur ForthEx et la REMOTE qui utilise le port sériel RS-232
pour se connecter à un PC qui utilise à émulateur de terminal VT102.
On passe d'une console à l'autre en invoquant leur nom:
REMOTE \ l'interface utilisateur utilise le port sériel.
LOCAL \ l'interface utilisateur utilise le clavier et le moniteur de l'ordinateur.
La variable système SYSCONS contient le vecteur de la console sélectionnée.
Ce vecteur est une table contenant les fonctions à exécuter pour chacune des
fonctions suivantes:
fonction#nom
0KEY
1KEY?
2EKEY
3EKEY?
4EMIT
5EMIT?
6AT-XY
7CLS
8XY?
9B/W
10INSRTLN
11RMVLN
12DELLN

REMOTE ( -- )

Passe en mode console VT102 via le port sériel.
Dans ce mode l'interface utilisateur utilise un terminal ou émulateur VT102
pour contrôler l'ordinateur.
La commication est à 115200 bauds, 8 bits, 1 stop bit et pas de parité.
Le contrôle de flux est logiciel via XON, XOFF.
L'ordinateur ForthEx n'implémente que partiellement le standard VT102
juste ce qui est nécessaire pour que la console REMOTE est les même fonctionnalités
que la console LOCAL.

arguments:
aucun   
retourne:
rien

KEY ( -- c )

Attend indéfiniement la réception d'un caractère de la console.
Les caractères invalides sont rejetés jusqu'à la réception
d'un caractère valide. Les caractères valides sont les suivants:
codetouche
virtuelle		fonction
4CTRL_DSupprime la ligne du curseur.
8VK_BACKEfface le caractère à gauche du curseur.
12CTRL_LEfface tout l'écran.
13VK_CRRetour à la ligne.
22CTRL_VEn interactif rappelle la dernière ligne.
32..126Caractères ASCII imprimables.

Pour accepter tous les codes du clavier il faut utiliser EKEY.

arguments:
aucun   
retourne:
c   Caractère reçu de la console

EMIT ( c -- )

Imprime les caractère ASCII dans l'interval {32..126} et accepte certains
codes de contrôle.Les codes de contrôles reconnus sont mis en actions,
les autres sont jetés. Les codes reconnus sont les suivants:
codetouche
virtuelle		fonction
4CTRL_DEfface la ligne du curseur sans fermer l'espace.
8VK_BACKSupprime le caractère à gauche du curseur.
9VK_TABAvance le curseur à la prochaine colonne.
12CTRL_LEfface l'écran.
13VK_CRRetour à la ligne.
24CTRL_XSuppression de la ligne du curseur et referme l'espace
en décalant vers le haut les lignes sous le curseur.
25CTRL_YInsertion d'une ligne en décalant vers le bas
les lignes à partir du curseur.
127VK_DELETESupprime le caractère à la position du curseur.
141VK_UPDéplace le curseur d'une ligne vers le haut.
142VK_DOWNDéplace le curseur d'une ligne vers le bas.
143VK_LEFTDéplace le curseur d'un d'un caractère vers la gauche.
144VK_RIGHTDéplace le curseur d'un caractère vers la droite.
145VK_HOMEDéplace le curseur au début de la ligne.
146VK_ENDDéplace le curseur à la fin de la ligne.
149VK_INSERTOuvre un espace à la position du curseur.

arguments:
c   Caractère à transmettre.
retourne:
rien
